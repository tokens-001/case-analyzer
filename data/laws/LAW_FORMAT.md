# 法条文件格式规范

## 文件命名

`<法律名>-<编章>.txt` 或 `<法律名>.txt`

例：`民法典-侵权责任编.txt`、`刑法.txt`

## 文件结构

```
#META
法名: 中华人民共和国民法典
编章: 第七编 侵权责任（如无编章可省略）
颁布日期: 2020-05-28
施行日期: 2021-01-01
最后修订: 2020-05-28
状态: 现行有效 | 已修订 | 已废止
取代: 被取代的法律名称（已废止/已修订时填写）
#END

第X条
条文内容（可跨多行，以空行分隔下一条）

第Y条
条文内容
```

## 元数据字段说明

| 字段 | 必填 | 说明 |
|---|---|---|
| 法名 | ✅ | 法律全称 |
| 编章 | 否 | 民法典分编时填写 |
| 颁布日期 | ✅ | YYYY-MM-DD |
| 施行日期 | ✅ | YYYY-MM-DD |
| 最后修订 | ✅ | 最近一次修订的日期 |
| 状态 | ✅ | 现行有效 / 已修订 / 已废止 |
| 取代 | 否 | 如该法取代了旧法，写明旧法名称 |

## 条文格式

- `第X条` 顶格写，条文内容紧跟其后
- 条文可跨多行
- 条文之间用空行分隔
- `#META...#END` 块内的内容不会被当作条文检索

## 程序读取逻辑

1. 读取文件 → 解析 `#META...#END` 获取元数据
2. 用 `全文.find("第X条")` 定位条文
3. 前端展示时标注状态颜色：绿色=现行有效，黄色=已修订，红色=已废止
