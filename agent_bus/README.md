# Agent Bus — AI 协作总线

## 原理

三个角色通过一个共享文件通信，各写各的槽位：

```
你（产品负责人）：写目标 + 做最终判断
GPT（战略顾问） ：写分析 + 写建议
Claude（执行工程师）：写实现 + 写阻塞点
```

通信协议是文件系统。不需要 API，不需要 WebSocket。

## 任务文件格式

每个任务一个 `.md` 文件，分五个槽位：

```
【目标】       ← 你写
【GPT分析】    ← GPT 写
【Claude执行】 ← Claude 写
【GPT评审】    ← GPT 写（可选）
【最终决策】   ← 你写
```

## 规则

1. 每个角色只写自己的槽位，不碰别人的
2. 写完一个槽位后，在文件名加状态标记（如 `T001_讨论中.md` → `T001_待执行.md`）
3. 你不需要参与每一轮——只需要在目标不清楚时澄清，在最终决策时拍板

## GPT 评审流程

Claude 有文件系统权限，GPT 没有。需要你手动桥接一次：

1. Claude 写入评审请求到 `GPT_REVIEW_QUEUE.md`
2. 你上传 `GPT_REVIEW_QUEUE.md`（或粘贴内容）给 GPT
3. GPT 写入评审结果到"GPT评审结果"槽位
4. 你复制回来保存
5. Claude 下次启动读 session-startup.md → 读 agent_bus/ → 自动看到 GPT 评审

你只动手一次，不需要理解内容。两个 AI 通过共享文件通信。
