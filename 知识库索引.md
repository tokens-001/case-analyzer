# 奇点知识库索引

> 这不是笔记库。这是一个个人研发组织的运作系统。
> 每个目录是一个职能部门，知识按固定方向流动。

---

## 目录地图

```
奇点/
│
├── 知识库索引.md          ← 你在这里
│
├── CLAUDE.md              ← 战略层：我是谁、怎么思考、行为准则
├── context/               ← 制度层：按场景触发的规则（启动/归档/备份）
├── agent_bus/             ← 协作层：你+GPT+Claude的共享任务板
│
├── knowledge/             ← 世界观层：14篇核心思考案卷 + 体系边疆索引
│   内容：长期积累的个人认知
│   更新：有新认知时写入
│
├── case_studies/          ← 案例复盘层：完整项目的事后拆解
│   内容：项目事实 + 关键决策 + 可复用规律
│   更新：每个项目完成后写一份
│
├── learnings/             ← 经验层：项目中发现的新规律（原料）
│   内容：发现 + 为什么重要 + 如何验证
│   状态：已升级5条→patterns，3条meta留存
│   更新：有新顿悟时写入
│
├── experiments/           ← 验证层：假设→设计→结果→状态
│   内容：实验假设、方法、结论、关联规律
│   更新：每做一个对比实验时写入
│
├── patterns/              ← 规律层：被项目验证过的可执行规律
│   内容：陈述 + 验证项目清单 + 反例 + 适用范围
│   更新：两个以上项目验证同一规律时从learnings升级
│
├── sessions/              ← 历史层：每次会话的推进轨迹
│   内容：对话推动的轨迹（非概念列表）
│   更新：每次会话结束时自动归档
│
└── python/                ← 生产层：判例助手 v3
    内容：Flask后端 + 前端 + skills/legal/13模块
```

## 使用说明

- **开始新会话**：读 `context/session-startup.md` → 读最新 session 归档
- **记录新发现**：写入 `learnings/`，三行格式（发现/为什么重要/如何验证）
- **做实验**：写入 `experiments/`，标注假设、方法、结果、状态
- **升级规律**：同一规律被两个项目验证后，从 learnings 迁移到 `patterns/`
- **项目复盘**：项目完成后写入 `case_studies/`，三层格式
- **认知沉淀**：长期认知写入 `knowledge/`，较短的直接放 learnings
