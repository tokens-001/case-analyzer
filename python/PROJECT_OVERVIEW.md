# 判例助手 —— 项目全貌 v3

## 定位

不是通用AI聊天工具，是**专精法律判例分析工具**。核心竞争力不在"接了什么模型"，而在**AI之外的硬校验层**——法条有效性、溯源可靠性、判例关联网络、反例发现——这些是通用AI做不到的事。

产品定位已从"帮律师找案例"升级为"**法律AI审计员**"——不是比谁分析得多，而是比谁能证明自己可信、能主动指出哪里不可信。

技术栈：Python Flask + DeepSeek API + 原生HTML/CSS/JS，部署在 PythonAnywhere。

## 已实现功能（按层次）

### 第一层：输入与校验（入口层）

| 功能 | 说明 |
|---|---|
| 判例文字粘贴/上传 | textarea粘贴 + 上传txt自动填充文件名 |
| 判例名称必填 | 前后端双重校验，空名称拒绝，最多80字 |
| 案发日期必填 | HTML date选择器 + required，后端验证 |
| 清空按钮 | 一键重置所有输入 |

### 第二层：AI分析（九维并行）

**基础四维（第一轮并行，ThreadPoolExecutor 4路）**

| 维度 | 说明 |
|---|---|
| 🎯 核心争议 | 案件的核心法律争议焦点分析 |
| 🧠 推理链路 | 法院从事实到结论的推理过程 |
| ❓ 未回答问题 | 判决中未解决的法律问题 |
| 🔀 可平移性 | 分析框架可推广到哪些法律领域 |

**深度五维（第二轮并行，ThreadPoolExecutor 5路，依赖基础四维结果）**

| 维度 | 说明 |
|---|---|
| 📋 综合总结 | 四段分析整合为一段完整总结（≤300字） |
| ⚔️ 反例检索 | 找出相反司法观点、被上诉攻击点、三个最强反驳论点 |
| 📐 判决结构化 | 事实认定→争议焦点→裁判理由→裁判结果四段式拆解 |
| 🔗 论证链检测 | 事实→法条→结论映射检测；未证明前提；替代论证路径 |
| 🔎 证据缺失检测 | 缺少的直接证据、最难举证环节、可改变判决的补充证据 |
| ⚖️ 相反法条发现 | 找出可能限制/冲突/抵消本案适用法条的其他条文 |

所有提示词硬性要求引用《XX法》第X条，至少2条。发送前统一插入`[第X段]`编号标记。

### 第三层：硬校验层（通用AI做不到的）

**溯源校验**

| 功能 | 说明 |
|---|---|
| 智能分段 | 空行→换行→句号三级自动拆分 |
| 溯源对比 | AI必须标注(见原文第X段)，代码用同一套分段函数校验，越界标红 |
| 编造检测 | 段落号超出原文范围自动判定"AI可能编造引用" |

**法条校验**

| 功能 | 说明 |
|---|---|
| 法条统计 | 通用正则匹配任意《XX法》第X条 |
| 法条库对照 | 从 data/laws/ 读取法条原文与AI引用逐条对照 |
| 法条有效性标签 | 元数据标注：现行有效✅(绿)、已修订⚠️(黄)、已废止❌(红) |
| 法条版本时间轴 | 案发日期 vs 法条施行/修订日期比对，版本不匹配标红警告 |
| 相邻法条发现 | AI引用法条的前后±2条自动扫描展示 |
| 缺失法条自增长 | AI引用过但法条库没有的，自动写入 missing_laws.txt |

**分析质量验证**

| 功能 | 说明 |
|---|---|
| 字数检查 | 每个维度字数过少判定API异常 |
| 法条引用检查 | 四段均未引用法条→告警 |
| 可信度评分 | 综合法条版本+溯源+验证，0-100分+高/中/低三级+来源明细 |

### 第四层：知识网络（壁垒层）

| 功能 | 说明 |
|---|---|
| 判例关联图谱 | 扫描全部历史判例→构建法条→判例反向索引 |
| 关联判例发现 | 查一个判例自动找引用相同法条的其他判例，按共同法条数排序 |
| 关联判例跳转 | 点击直接加载完整九维分析 |
| 历史记录 | 自动保存JSON，列表显示案件名+日期+总结摘要 |
| 历史搜索 | 关键词实时过滤判例名和总结 |
| 历史详情回看 | 点击加载完整九维分析（含关联判例） |

### 第五层：风险与可信度（优先展示）

| 功能 | 说明 |
|---|---|
| 风险列表 | 三级汇总（验证+法条版本+溯源越界），红色danger/黄色warning |
| 风险优先展示 | 打开结果最先看到"哪里可能错"，不是先看分析卡片 |
| 可信度仪表盘 | 圆环分数+等级文字+加减分明细来源 |

**可信度评分逻辑**：基础70分 + 法条版本一致+15 / 不匹配-10每条 + 法条有效+5 / 无法条库-10 + 溯源全过+10 / 越界-5每处 + 验证通过+5 / 告警-5每条。范围0-100。

### 第六层：输出与交互

| 功能 | 说明 |
|---|---|

| 综合总结卡片 | 绿色左边框独立展示 |
| 折叠区 | 溯源对比/法条库对照/关联判例默认折叠，点击箭头展开 |

| 结果渐入动画 | 依次入场，级联延迟 |
| 下载报告 | 一键导出13段完整txt（九维+法条统计+法条对照+溯源对比） |
| 暗色专业主题 | CSS变量系统，GitHub风格深色界面 |

## 核心理念

> AI加功能是横向加法，谁都能做。专精是纵向减法——砍掉泛的，把AI之外的硬校验做到别人没法复制。

四个壁垒：
- **法条时效性**：AI不知道哪条法废止了、版本对不对，我知道
- **溯源可验证**：AI会编造引用，我逐条对原文并标红越界
- **反例发现**：大多数AI工具只找支持你的案例，我找反对你的案例
- **判例关联网**：AI只看单案，我能串出法条→判例→裁判结果的关系网
