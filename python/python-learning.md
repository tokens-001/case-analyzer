# Python学习案卷

## 通用编程名词速查（学新概念前先翻）

### 程序怎么跑起来
- **源代码**：你写的代码文件（`.py`）。人能读，机器还没
- **编译/解释**：把源代码翻译成机器能执行的指令。Python 是解释型——一行一行翻译执行，不需要先编译
- **运行时（Runtime）**：程序执行的那个时刻。"运行时决定" = 不提前写死，跑的时候看情况
- **语法**：写代码的规则——缩进、引号。违反语法 = 跑不起来

### 代码里的零件
- **变量**：带名字的数据容器。`name = "小明"` → `name` 是变量名，"小明" 是值
- **函数**：一段有名字的可重复使用的代码。`def 函数名(参数):` 定义，`函数名(值)` 调用
- **参数 vs 返回值**：参数 = 函数吃进去的，返回值 = 函数吐出来的。`return` 后面的是返回值
- **字符串**：文字数据，用引号包起来。`"hello"`
- **整数/浮点数**：`42`（整数），`3.14`（浮点数/小数）
- **布尔值（bool）**：`True` 或 `False`。条件判断用——"这段代码跑不跑？"
- **列表（List）**：有序的一组数据。`["苹果", "香蕉"]`。用位置序号 `[0]` 取
- **字典（dict）**：带标签的数据。`{"名字": "小明", "年龄": 25}`。用标签名 `["名字"]` 取
- **集合（Set）**：不重复的一组数据。自动去重——`{"苹果","苹果"}` → 只剩一个
- **键（Key）vs 值（Value）**：字典里冒号左边是键（标签），右边是值（内容）

### 代码怎么组织
- **模块（Module）**：一个 `.py` 文件。`import os` = 把 `os.py` 的功能搬过来用
- **库（Library）**：别人写好的功能集合。`requests` 库做网络请求，`json` 库处理 JSON
- **依赖（Dependency）**：你的代码用到别人的哪些库。没装 `requests` 就报错——它就是你的依赖
- **导入（Import）**：`import X` ——把外部模块/库的功能引入当前文件
- **API**：别人提供的功能接口。DeepSeek API——你把问题发过去，它返回答案。你不需要知道它内部怎么算
- **SDK**：一套包装好的工具/库/文档合集，方便你调用某个平台的功能

### 数据怎么存和传
- **文件路径**：文件在电脑上的地址。`/Users/jingzhe/奇点/xxx.txt`（绝对路径）vs `data/xxx.txt`（相对路径）
- **JSON**：通用数据交换格式。`{"名字": "小明"}`。`json.dump()` 写，`json.load()` 读
- **序列化/反序列化**：Python 数据→字符串存盘 = 序列化。读回来变回 Python 数据 = 反序列化
- **持久化**：数据不随程序结束而消失。存到硬盘文件里 = 持久化

### 程序怎么控制流程
- **条件判断**：`if/elif/else`——如果X就A，否则如果Y就B，都对不上就C
- **循环（Loop）**：重复执行。`while True:` 一直循环到 `break`。`for 变量 in 列表:` 遍历每个元素
- **迭代/遍历**：一个一个过。`for 文件名 in 文件列表:` 遍历每个文件
- **break vs continue**：`break` = 跳出整个循环。`continue` = 跳过本次，进下一轮
- **递归**：函数自己调用自己。搞不好就死循环

### 程序出错了怎么办
- **Bug**：程序里的错误/缺陷
- **异常（Exception）**：运行时爆的错误。`try/except` 抓异常，防程序直接死掉
- **调试（Debug）**：找 bug 并修的过程。`print()` 看变量值是最原始的调试
- **语法错误 vs 运行时错误**：语法错 = 写都写错了跑不起来。运行时错 = 语法对但跑的时候炸了
- **缩进**：Python 里空格不是装饰，是语法——缩进定义代码属于哪个 `if`/`while`/`def` 块

### 代码怎么管理
- **Git**：版本控制系统。每次 `commit` = 一个快照，以后可以回退
- **仓库（Repository/repo）**：一个项目的 Git 存储空间
- **Commit**：把改动保存到 Git 历史
- **Staging**：暂存区。`git add` 选文件放进去→`git commit` 一次性提交

### 网页开发基础名词
- **前端（Frontend）**：浏览器里跑的——HTML+CSS+JS。用户能看见的
- **后端（Backend）**：服务器上跑的——Python+Flask 处理请求。用户看不见
- **服务器**：一直开着的电脑，等着处理请求
- **客户端**：用户的浏览器。发请求→收响应
- **路由（Route）**：网址→函数的映射。`/analyze` 网址 → 调分析函数。类似菜单"选1"
- **HTTP 请求/响应**：客户端说"我要这个"（请求）→ 服务器给答案（响应）
- **模板（Template）**：HTML 骨架，空槽由后端填充数据
- **数据库（Database）**：结构化的持久化存储。你现在用 JSON 文件，大型项目用 MySQL/PostgreSQL

### 进阶概念（已接触）
- **正则表达式（Regex）**：按模式匹配文字的工具。`\d+` = 一个或多个数字。`re.findall()` 从文字里抓
- **捕获组 vs 非捕获组**：`( )` 配合 `.group(1)` 提取内容。`(?: )` 只匹配不提取
- **RAG**：检索增强生成。先搜资料，把资料+问题一起给 AI，提高准确度
- **Embedding**：文字→数字向量。"距离"≈"语义相关度"。RAG 的基础
- **Agent（智能体）**：能自主决策下一步的 AI 系统，不只回答问题
- **Deploy（部署）**：代码搬到服务器让它跑起来

## 第一节：第一句话
- `print("内容")` — 把括号里的东西输出到屏幕
- `"内容"` 叫字符串，用引号包起来

## 第二节：让程序问问题
- `input("提示文字")` — 输出提示，停下来等人输入，返回输入的内容
- 变量 — 把数据存起来，比如 `name = input("你是谁？")`，之后用 `name` 就能拿到输入的内容
- `+` — 把字符串拼在一起

## 第三节：判断
- `if 条件:` — 如果条件成立，执行下面缩进的代码
- `elif 条件:` — 否则如果另一个条件成立
- `else:` — 以上都不成立时执行
- `==` — 判断两边是否相等（一个 `=` 是赋值，两个 `==` 才是比较）
- **缩进必须一致**，Python 靠缩进识别代码块
- ✅ 已实践：guess.py 三次测试通过

## 第三节实践补充：input()
- `name = input("提示")` — 把用户输入的内容存到变量 name 里
- `print("你好，" + name)` — 用 + 拼接字符串和变量
- ✅ 已实践：ask.py 测试通过

## 第四节：循环
- `while 条件:` — 条件成立就一直重复执行缩进的代码块
- 循环变量需要在 while 之前初始化（如 `count = 1`）
- 循环体内必须改变条件变量（如 `count = count + 1`），否则死循环
- ✅ 已实践：loop.py 测试通过

## 第五节：转义符与repr
