## 后端 `app.py` (≈650行)

```
app.py
│
├── 顶部 ── import 依赖、Flask初始化、.env加载、路径变量
│   ├── 加新依赖库 → import 区域
│   ├── 改端口号 → 最后一行 port=5050
│   └── 改环境变量读取 → 项目根/法条库目录/数据根目录

│   └── 智能分段()

│   └── 问AI() → 插入[第X段]编号后发送请求

│   ├── 法条提示 变量 → 硬性要求引用法条
│   ├── 核心争议()
│   ├── 推理链路()
│   ├── 未回答问题()
│   ├── 可平移性()
│   ├── 案例总结() → 整合四段分析
│   ├── 反例检索() → 找反对观点和薄弱环节
│   ├── 结构化摘要() → 事实→争议→理由→结果四段式
│   ├── 论证链检测() → 找逻辑断裂和未证明前提
│   ├── 证据缺失检测() → 缺少哪些关键证据
│   └── 相反法条发现() → 找可能冲突/抵消的法条
│
├── #4 溯源对比 ── AI引用 vs 原文对照
│   └── 溯源对比() → 校验段落号，越界标红
│
├── #5 法条统计 ── 通用正则抓取法律条文引用
│   └── 法条统计() → 匹配《XX法》第X条
│
├── #6 法条库查询 ── 从data/laws/找条文原文
│   ├── 解析法条元数据() → 读取#META信息

│   └── 查关联判例() → 找引用相同法条的其他判例
│
├── #6c 法条版本时间轴校验

│   ├── 生成风险列表() → 验证+版本+溯源三级汇总
│   ├── 相邻法条发现() → 引用法条附近±2条原文
│   └── 计算可信度() → 0-100分+高/中/低等级+来源明细
│
├── #7 验证 ── 分析质量检查
│   └── 验证分析结果()
│
├── #8 存储 ── 存JSON到本地
│   └── 存储为JSON() / _存储为JSON_内部() → 9个分析维度全存
│
├── #9 每日次数限制 ── 防滥用
│   ├── 获取客户端IP() → X-Forwarded-For优先
│   ├── 剩余次数查询() → 仅查询不消耗
│   └── 消耗次数() → UID+IP双轨
│
└── 路由 ── URL到函数的映射
    ├── /            → 首页
    ├── /analyze     → POST 分析主流程（频次→分段→9维并行→校验→图谱→可信度→返回）

    ├── /history     → GET 历史列表
    ├── /case/<fname>→ GET 单条详情+关联判例
    ├── /related/<fname>→ GET 关联判例
    └── /download    → POST 生成下载文本
```

### 后端数据流（一次分析的完整路径）

```
用户粘贴文字+案发日期 → /analyze路由
  → 检查次数(UID+IP双轨)
  → 智能分段(判例) → 插入[第X段]编号
  → ThreadPoolExecutor并行4路 → 核心争议/推理链路/未回答问题/可平移性
  → 案例总结(依赖上面4个)
  → ThreadPoolExecutor并行5路 → 反例检索/结构化摘要/论证链检测/证据缺失检测/相反法条发现
  → 溯源对比(原文 vs AI引用)
  → 法条库查询 + 缺失记录
  → 法条版本校验(案发日期 vs 施行日期)
  → 相邻法条发现
  → 验证分析结果
  → 计算可信度(0-100分)
  → 生成风险列表
  → 存储为JSON
  → return jsonify({分析9维, 溯源, 法条, 相邻法条, 验证, 可信度, 风险列表, 剩余次数})
```
