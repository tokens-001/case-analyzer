# 判例助手 功能树 v3（2026-06-13 更新）

## 定位

专精法律判例分析的AI校验引擎。核心壁垒不在模型，在AI外层的硬校验层——法条有效性、溯源可验证、判例关联网络。

技术栈：Python Flask + DeepSeek API + 原生HTML/CSS/JS（零前端框架），部署在 PythonAnywhere。

代码规模：app.py ≈830行，index.html ≈910行。

---

## 第一层：输入与校验（入口层）

| 功能 | 状态 | 说明 |
|---|---|---|
| 判例文字粘贴/上传 | ✅ | textarea粘贴 + 上传txt文件自动填充 |
| 判例名称必填 | ✅ | 前后端双重校验，空名称拒绝，最多80字 |
| 案发日期必填 | ✅ | HTML date选择器 + required属性，后端验证 |
| 文件上传自动填名 | ✅ | 上传txt时自动用文件名填判例名称 |

| 清空按钮 | ✅ | 一键重置所有输入 |

## 第二层：AI分析（九维并行分析）

### 基础四维（第一轮并行）
| 维度 | 标识 | 说明 |
|---|---|---|
| 核心争议 | 🎯 | 案件的核心法律争议焦点分析 |
| 推理链路 | 🧠 | 法院从事实到结论的推理过程 |
| 未回答问题 | ❓ | 判决中未解决的法律问题 |
| 可平移性 | 🔀 | 分析框架可推广到哪些领域 |

### 深度五维（第二轮并行，依赖基础四维结果）
| 维度 | 标识 | 说明 |
|---|---|---|
| 综合总结 | 📋 | 四段分析整合为一段完整总结（≤300字） |
| 反例检索 | ⚔️ | 找出相反的司法观点、被上诉攻击点、三个最强反驳论点 |
| 判决结构化 | 📐 | 四段式结构化：事实认定→争议焦点→裁判理由→裁判结果 |
| 论证链检测 | 🔗 | 事实→法条、法条→结论映射检测；未证明前提；替代论证路径 |
| 证据缺失检测 | 🔎 | 缺少的直接证据、难以举证的环节、改变判决的关键补充证据 |
| 相反法条发现 | ⚖️ | 找出可能限制/冲突/抵消本案适用法条的其他条文 |

### 技术实现
- ThreadPoolExecutor 两轮并行（4路→5路），总耗时约35-55秒
- 所有提示词强制要求引用《XX法》第X条，至少2条
- 发送前插入`[第X段]`编号标记，保证AI引用与代码拆分一致

## 第三层：硬校验层（AI做不到的）

### 溯源校验
| 功能 | 说明 |
|---|---|
| 智能分段 | 空行→换行→句号三级自动拆分，三等分时标注 |
| 原生溯源对比 | AI必须标注(见原文第X段)；代码用同一套分段函数校验；越界引用标红警告；统计引用数/总段落数 |
| 编造检测 | 段落号超出原文范围的自动判定为"AI可能编造引用" |

### 法条校验
| 功能 | 说明 |
|---|---|
| 法条统计 | 通用正则匹配任意《XX法》第X条，不限特定法律 |
| 法条库对照 | 从 `data/laws/` 读取法条原文，与AI引用逐条对照 |
| 法条有效性标签 | 元数据标注：现行有效✅(绿)、已修订⚠️(黄)、已废止❌(红) |
| 法条版本时间轴 | 案发日期 vs 法条施行/修订日期比对；案发早于施行→警告可能适用旧法 |
| 相邻法条发现 | AI引用法条的前后±2条自动扫描展示（前条蓝色标签/后条紫色标签） |
| 缺失法条自增长 | AI引用过但法条库没有的，自动写入 `missing_laws.txt`，指导后续扩充 |

### 分析质量验证
| 功能 | 说明 |
|---|---|
| 字数检查 | 每个维度字数过少判定API异常 |
| 法条引用检查 | 四段均未引用法条→告警 |
| 可信度评分 | 综合法条版本、溯源准确性、验证结果，算出0-100分+高/中/低三级 |
| 可信度仪表盘 | 圆环分数+等级文字+加减分明细来源 |

## 第五层：风险与可信度（最先展示）

### 风险列表生成
- 来源一：验证层风险（字数不足、未引用法条）→ warning级别（黄色）
- 来源二：法条版本风险（案发时间早于施行日期、已废止、已修订）→ danger级别（红色）
- 来源三：溯源越界风险（AI编造不存在的段落号）→ danger级别（红色）
- 结果区最优先展示，排在所有分析卡片之前

### 可信度评分逻辑
| 评分项 | 影响 |
|---|---|
| 基础分 | 70 |
| 法条版本全部一致 | +15 |
| 法条版本不匹配（每条） | -10（最多-20） |
| 法条有库中对照且现行有效 | +5 |
| 无法条库对照 | -10 |
| 溯源引用全部在原文范围内 | +10 |
| 溯源越界（每处） | -5（最多-15） |
| 验证通过 | +5 |
| 质量告警（每条） | -5 |
| 总分范围 | 0-100 |
