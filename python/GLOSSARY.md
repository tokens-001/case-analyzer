## 六、正则表达式

| **Hallucination 幻觉** | AI一本正经地编造不存在的事实 | 你的溯源对比就是防这个——段落号越界=AI编的 |
| **Fine-tuning 微调** | 用自己的数据再训练一下模型，让它更懂你的领域 | 你没用，你用的是提示词工程（成本更低） |

## 十二、AI应用架构

| 词 | 一句话 | 在你项目里 |
|---|---|---|
| **API调用** | 你的程序通过网络发请求给AI服务，拿回结果 | `问AI()` 函数做的就是这个 |
| **并行调用** | 同时发多个请求给AI，不等上一个回来就发下一个 | `ThreadPoolExecutor` 同时跑4个分析 |
| **Agent 智能体** | 让AI不仅回答问题，还能调用工具（查数据库、发邮件、执行代码） | 你的判例助手还不是Agent——它只生成文字，不自己操作外部工具 |

| **RAG 检索增强生成** | 先搜索相关文档，再把文档内容拼进提示词让AI参考——减少幻觉 | 你的法条库查询就是简易版RAG：先本地搜法条→再把原文展示给用户 |
| **Embedding 向量嵌入** | 把一段文字变成一串数字（向量），相似内容的向量也相似——用来做语义搜索 | 你还没用到，以后升级检索可以用 |
| **Vector DB 向量数据库** | 存储和搜索向量的专用数据库 | 你没用，你用的是文件系统+关键词匹配 |
| **Orchestration 编排** | 把多个AI调用串联成一条流水线 | 你的分析流程就是编排：4个分析→总结→溯源→法条→存储 |
| **Guardrail 护栏** | 限制AI不乱说的规则——防止输出敏感、违法、或无关内容 | 你的验证函数和次数限制都是护栏 |
| **Rate Limit 速率限制** | 限制调用频率——防刷、防烧钱 | 你的每日3次就是rate limit |
| **Fallback 兜底** | 主方案失败时的备用方案 | `try...except` 就是代码层面的兜底 |

## 十三、推理与思维链

| 词 | 一句话 | 在你项目里 |
|---|---|---|
| **Chain of Thought 思维链** | 让AI在给最终答案前先"一步一步推理"——准确率明显提升 | 你的四维分析暗含这个：推理链路、核心争议都是分步骤的 |
| **Zero-shot** | 不给例子，直接让AI做一件事 | 你的提示词就是zero-shot："请分析核心争议" |
| **Few-shot** | 给AI一两个例子，再让它做类似的事——效果更好 | 你没用，但加一两个判例分析样例就能提升质量 |
| **Reasoning Model 推理模型** | 专门强化过推能力的模型——会自己内部"想"一段时间再输出 | DeepSeek-R1就是，你现在用的deepseek-chat不是 |
| **Thinking Token** | 推理模型内部思考时消耗的token——你能看到推理过程，但不计入最终输出 | Claude Opus的`<thinking>`块就是这个 |
