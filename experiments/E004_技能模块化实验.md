# E004 — 技能模块化 vs 单体架构实验

## 假设

把 AI 产品的法律逻辑从 Web 框架中抽离成独立技能模块，能提升复用性且不破坏现有功能。

## 来源

判例助手 v3 架构重构

## 实验设计

- 对照组：app.py 880+ 行，所有分析函数 + 校验函数 + 路由全部在一个文件
- 实验组：抽离 13 个独立技能模块到 skills/legal/，app.py 减至 532 行，/analyze 变为编排层
- 关键约束：前端零改动，7 个路由全部兼容

## 结果

- 兼容性：✅ 全部 7 个路由自动化测试通过，前端一行未改
- 代码精简约 30%（880 → 532 行）
- 每个技能现在可独立 import 调用
- 未来新场景（CLI/Agent/其他产品）可直接复用技能模块

## 状态：✅ 验证成功（n=1）
