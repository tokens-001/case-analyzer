## 一句话
模型负责聪明，框架负责纪律。AI 能力够强但不稳定，稳定性必须由外部框架供给。

## 五层架构

### 1. 常驻入口层（每轮都在，≤8K）
CLAUDE.md + CLAUDE.local.md：角色定义 + 代码偏好 + 流程触发规则 + 门禁速查。自包含，不依赖 @import。

### 2. 原子规则层（按需引用）
7条规则，每条=一次事故的墓志铭。踩坑→固化成规则→永久兜底。

### 3. 角色 Agent 层（dispatcher call_agent 激活）
核心设计：主会话退化成纯执行器。dispatcher（交警）只读 state.json 算下一步。各 agent 从干净上下文启动、文件交接（A写→B读）。职责隔离杜绝上下文污染。

### 4. 按需上下文层（进阶段才加载，用完释放）
深度内容（TDD指南、辩论模板、证据链规范）只在该阶段 Read。常驻层始终精简。

### 5. 执行支撑层
skills/ + commands/ + evals/：工具链封装。经验三级进化：lesson→pattern→instinct。

## 门禁系统
G1-G8 确定性 Python 检查——产物在不在、编译过没过、单测通没通。任一 FAIL 则流程退回。不是"建议"，是"阻断"。hooks 在工具调用前拦截。

## 关键数据
- token 预算对成功率的解释力 R²=0.33~0.42
- 验证反馈质量对成功率的解释力 R²=0.94~0.99
- 结论：检查比预算重要

## 编排机制选择
| 机制 | 适用 | 短板 |
|------|------|------|
| Workflow（JS脚本） | 计算平面：高并行单阶段 | 超时杀进程、无人工交互、跨session不可续 |
| Agent Team | 协作平面：多人独立任务 | 松散协调、状态散落、无法硬围栏 |
| Dispatcher+文件交接 | 控制平面：有状态工序链+人工门禁+跨天 | token IO开销、调试链路长 |

## 演进四阶段
拿来主义→重prompt约束（崩了）→减负+分层加载→Agent调度+精简。每一跳的拐点都是"现在的做法已经崩了"。

## 评测平台
7维确定评分（流程完整性22%、代码正确性22%、接口验收18%、产物质量15%、效率10%、合规8%、迭代5%）。零LLM调用，3次跑分hash完全一致。用分数回答"改完到底变好还是变坏"。
