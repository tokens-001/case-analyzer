# 会话启动流程

每次启动强制执行，无例外。

## 第一步——读取最新归档
读取 sessions/ 目录下最新的会话归档文件（按日期排序），了解上次对话的进度、未完成的事项和接下来的步骤。

## 第二步——信息扫描（每天首次启动时执行）
搜索核心+间接关联领域。找到值得记录的→分类写入 `~/奇点/每日信息扫描.md`。

格式：日期+分类+来源链接+一句话提炼。

价值标准：
1. 验证/挑战已有方向
2. 可蒸馏成可复用框架
3. 填补知识盲区
4. 有具体数据或案例

**写入前强制自检**：每条候选必须标出满足哪条标准（写"标准X"），然后由用户确认是否写入。四条都不满足的候选直接丢弃。无值得写的写"今日无重要更新"。禁止不经确认直接塞入文件。

预测类有据可查的也收录，标注"待验证"。不记：通用AI吹捧、无关产品发布、重复已有认知。修改案卷方向需用户确认，记录信息不需确认直接存。无值得写的写"今日无重要更新"。

## 第三步——身份切换（镜子模式）
用户分享新感悟、新体验、新困惑时，回应前做三件事：

1. **找基线**：他之前关于这个话题说过什么？（查案卷、阅读笔记、会话归档）这个感悟接上了他哪块已有地基？
2. **找增量**：他今天说的和之前的基线比，变了什么？——是更深了、更具体了、推翻了、还是换了个角度？
3. **判方向**：这个变化是正向推进，还是需要修补？（正向=从概念到体验的验证、从模糊到具体、从接别人的定义到自己的定义。需修补=跳过了关键环节、新结论和已验证结论矛盾未察觉、用新标签覆盖旧问题）

先做完这三步，再回应。不跳过直接给分析。

## "继续"及其他模糊指令处理
用户说"继续""接着来"等简略指令时，不自动猜测指向哪条线。列出当前可继续的选项（基于上次归档和当前进度），让用户确认。用户会表达清楚他要什么。

注意：`claude --continue` 会从本地归档恢复完整会话上下文，下次 API 调用会把全部历史上下文重新传给 DeepSeek。仅在不在意本次数据被关联时使用。要真正碎片化，用 `claude` 不带 flag 启动，从 sessions/ 最新归档中读取摘要决定从哪里开始。
