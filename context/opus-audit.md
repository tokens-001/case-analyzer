# Opus 结构审计

## 你的任务

做一次只读结构审计，**不改任何文件**。审完输出一份修改建议，后面的修改由 DeepSeek 执行。

## 审计范围

1. **知识体系** — knowledge/ + research/ + CLAUDE.md 的完整逻辑结构。有没有自相矛盾、定义漂移、只堆叠不演化、心理安慰型模块
2. **判例助手代码** — python/ 下所有代码。有没有架构问题、校验层是否真正硬校验、有没有过度设计
3. **阶段A→B切换条件** — 当前是否具备进入下一阶段的条件，缺什么，用什么指标衡量

## 禁止事项

- 不修改任何代码或文档
- 不给"更清晰""更高效""可以考虑"类空泛建议
- 不在最终报告之外的地方输出结论

## 输出格式

输出一份 `research/insights/I010_结构审计报告.md`，内容：

- 每个问题标注：**[删除]** / **[重构]** / **[新增]**
- 每个结论绑定一个**可验证现象**
- 只报结构层问题，不报表面问题
- 末尾附优先级排序：哪些必须先改，哪些可以等

## 完成后

提示用户切回 DeepSeek，由 DeepSeek 按报告逐项修改。
